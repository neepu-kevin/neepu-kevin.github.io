GitHub 高级搜索：从 in:readme 开始精准找项目

2026-07-04 | 日常使用 | GitHub, search

GitHub 的搜索框不只是用来搜项目名。只要给关键词加上一些限定条件，就可以把搜索范围、项目热度、语言、更新时间等条件组合起来，快速找到更接近目标的仓库。

这篇文章先从最常用的 in:readme 讲起。

用 in 限定关键词出现的位置

默认搜索仓库时，GitHub 主要会匹配仓库名称、描述和 topics。如果想明确指定搜索范围，可以使用 in:。

常见写法：

  • xxx in:name：项目名包含 xxx
  • xxx in:description：项目描述包含 xxx
  • xxx in:readme：README 中包含 xxx
  • xxx in:topics：topics 中包含 xxx
  • xxx in:name,description,readme：同时搜索项目名、描述和 README

如果你正在找教程、模板、配置示例，in:readme 往往比只搜项目名更好用。因为很多项目名并不包含关键词，但 README 会写清楚它解决什么问题。
